## 核心概念

### 定义

客户端负载均衡从注册中心获取实例列表，按策略选择节点。Spring Cloud 2020+ 默认使用 Spring Cloud LoadBalancer。

### 常见误区

- 仍寻找 `spring-cloud-starter-ribbon`（已移除）
- 未注册 Eureka 却使用 `lb://` 或 `@LoadBalanced`
- 实例全下线导致 `No instances available`

## 子项目规格

| 字段 | 值 |
|------|-----|
| **moduleId** | `sc-loadbalancer-demo` |
| **path** | `springcloud/sc-loadbalancer-demo/` |
| **port** | 8085（实例 2 可用 8089） |
| **entry** | `com.lean.lb.LoadBalancerDemoApplication` |

### 结构说明

本模块同时包含：

- **Provider** 逻辑：多实例启动同一 `spring.application.name: lb-demo-service`，`GET /instance` 返回当前端口
- **Consumer** 逻辑：同模块或拆分为两个启动类（阶段二建议拆 `provider`/`consumer` 两个 main 配置类，文档阶段用两个 profile 启动两次 provider + 一个 consumer）

简化方案（文档约定）：

| 子目录角色 | 说明 |
|------------|------|
| 启动 profile `provider` | 暴露 `/instance` |
| 启动 profile `consumer` | `@LoadBalanced` 调用 `http://lb-demo-service/instance` |

### application-provider.yml

```yaml
server:
  port: 8085
spring:
  application:
    name: lb-demo-service
```

### application-consumer.yml

```yaml
server:
  port: 8090
spring:
  application:
    name: lb-consumer
```

### 运行与验证

```bash
# Eureka 已启动
mvn spring-boot:run -Dspring-boot.run.profiles=provider
# 改 port 8089 再启第二个 provider
mvn spring-boot:run -Dspring-boot.run.profiles=consumer
curl http://localhost:8090/call
```

多次请求 `call` 应看到不同实例端口（轮询）。
