## 核心概念

### 定义

消息中间件缓冲生产与消费速率差异，实现系统解耦。RabbitMQ 是常用 AMQP 实现。

### 常见误区

- 无幂等消费导致重复处理
- 未 ACK 导致消息丢失或堆积
- 队列无 DLQ 处理 poison message

## 子项目规格

| 字段 | 值 |
|------|-----|
| **moduleId** | `sc-rabbitmq-demo` |
| **path** | `springcloud/sc-rabbitmq-demo/` |
| **port** | 8088 |
| **entry** | `com.lean.mq.RabbitMqDemoApplication` |

### Maven

```xml
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-amqp</artifactId>
</dependency>
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
</dependency>
```

### application.yml

```yaml
server:
  port: 8088
spring:
  rabbitmq:
    host: localhost
    port: 5672
    username: guest
    password: guest
```

### 包结构

| 类 | 说明 |
|----|------|
| `RabbitConfig` | 声明队列 `demo.queue` |
| `MessageProducer` | `POST /publish` 发消息 |
| `MessageListener` | `@RabbitListener` 消费并日志 |

### RabbitMQ 启动

```bash
docker run -d --hostname rabbit --name rabbit -p 5672:5672 -p 15672:15672 rabbitmq:3-management
```

### 运行与验证

```bash
cd springcloud/sc-rabbitmq-demo && mvn spring-boot:run
curl -X POST "http://localhost:8088/publish?body=hello"
```

控制台应出现消费日志。
