## 核心概念

### 定义

集中管理多环境配置，避免打包后改配置。Config Server 暴露 HTTP API，Client 启动时合并远程属性源。

### 常见误区

- 配置仓库含密钥明文提交 Git
- `spring.application.name` 与配置文件名不匹配
- 未区分 profile 导致拉错文件

## 子项目规格

### sc-config-server

| 字段 | 值 |
|------|-----|
| **path** | `springcloud/sc-config-server/` |
| **port** | 8888 |
| **entry** | `com.lean.config.ConfigServerApplication` |

#### Maven

`spring-cloud-config-server`

#### application.yml

```yaml
server:
  port: 8888
spring:
  application:
    name: sc-config-server
  profiles:
    active: native
  cloud:
    config:
      server:
        native:
          search-locations: classpath:/config-repo
```

#### 资源文件

`src/main/resources/config-repo/demo-client.yml`：

```yaml
message: "from config server"
```

### sc-config-client

| 字段 | 值 |
|------|-----|
| **path** | `springcloud/sc-config-client/` |
| **port** | 8082 |
| **entry** | `com.lean.config.ConfigClientApplication` |

#### application.yml

```yaml
server:
  port: 8082
spring:
  application:
    name: demo-client
  config:
    import: optional:configserver:http://localhost:8888
```

#### 接口

`GET /message` 返回配置中的 `message`

### 运行与验证

```bash
cd springcloud/sc-config-server && mvn spring-boot:run
cd springcloud/sc-config-client && mvn spring-boot:run
curl http://localhost:8082/message
```
