## 核心概念

### 定义

全局异常处理避免在每个 Controller 写 try-catch，将错误转换为稳定 JSON 格式与合适 HTTP 状态码。

### 常见误区

- 捕获 `Exception` 后返回 200 状态码
- 向客户端暴露堆栈与内部 SQL
- 多个 `@ExceptionHandler` 顺序不当导致永不命中

## 关键 API / 注解

| 名称 | 说明 |
|------|------|
| `@RestControllerAdvice` | 全局 REST Advice |
| `@ExceptionHandler` | 处理特定异常 |
| `@ResponseStatus` | 指定 HTTP 状态 |

| 类 | 说明 |
|----|------|
| `ApiResponse` | 统一响应 |

| `DemoController` | `GET /demo/error` 触发异常 |

### 运行与验证

```bash
cd springboot/sb-08-global-exception
mvn spring-boot:run
curl -i http://localhost:8080/demo/error
```

**期望：** HTTP 400，body 含 `code` 与 `message`，无堆栈。
