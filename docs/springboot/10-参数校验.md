## 核心概念

### 定义

声明式校验将规则写在 DTO 字段注解上，减少手写 if。Spring MVC 在入参绑定时触发校验。

### 常见误区

- 忘记 `spring-boot-starter-validation` 依赖
- 只校验字段不校验嵌套对象（需 `@Valid` 嵌套）
- 在 Service 重复相同校验

## 关键 API / 注解

| 名称 | 说明 |
|------|------|
| `@NotBlank` / `@Email` / `@Min` | 约束 |
| `@Valid` | 启用校验 |
| `MethodArgumentNotValidException` | 校验失败异常 |

## 子项目规格

| 字段 | 值 |

| **moduleId** | `sb-10-validation` |

| **entry** | `com.lean.validation.ValidationApplication` |

### Maven 依赖

```xml
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-validation</artifactId>
</dependency>
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
</dependency>
```

### 包结构

| 类 | 说明 |
|----|------|
| `UserCreateRequest` | `@NotBlank String name` 等 |
| `UserController` | `POST /users` |
| `ValidationExceptionHandler` | 处理校验异常 |

### 运行与验证

```bash
cd springboot/sb-10-validation
mvn spring-boot:run
curl -X POST http://localhost:8080/users -H "Content-Type: application/json" -d "{}"
```

**期望：** HTTP 400，返回字段错误列表。
