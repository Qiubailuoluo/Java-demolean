## 核心概念

### 定义

集成测试加载 Spring 容器验证协作；切片测试只加载 MVC 层更快。MockMvc 模拟 HTTP 请求断言状态码与 JSON。

### 常见误区

- 测试依赖执行顺序（应相互独立）
- 集成测试过多导致 CI 变慢
- 未使用 `@ActiveProfiles("test")` 污染开发库

## 关键 API / 注解

| 名称 | 说明 |
|------|------|
| `@SpringBootTest` | 全上下文测试 |
| `@AutoConfigureMockMvc` | 注入 MockMvc |
| `@MockBean` | Spring 上下文 Mock |
| `@WebMvcTest` | MVC 切片 |

### Maven 依赖

```xml
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
</dependency>
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-test</artifactId>
  <scope>test</scope>
</dependency>
```

### 包结构

| 类 | 说明 |
|----|------|
| `GreetingController` | `GET /greet` |
| `GreetingService` | 返回问候语 |
| `GreetingControllerTest` | `@WebMvcTest` + MockMvc |
| `GreetingServiceTest` | `@SpringBootTest` + `@MockBean` |

### 运行与验证

```bash
cd springboot/sb-14-testing
mvn test
```
