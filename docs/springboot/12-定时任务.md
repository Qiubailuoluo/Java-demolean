## 核心概念

### 定义

Spring 内置调度在单实例上运行定时方法。多实例重复执行需 Quartz、XXL-JOB 或 ShedLock 等。

### 常见误区

- 长任务阻塞调度线程（默认单线程池）
- cron 表达式时区未指定
- 定时任务内抛异常未捕获导致后续不执行

## 关键 API / 注解

| 名称 | 说明 |
|------|------|
| `@EnableScheduling` | 启用调度 |
| `@Scheduled` | 定时方法 |
| `fixedRate` / `fixedDelay` / `cron` | 触发策略 |

### Maven 依赖

`spring-boot-starter-web`（可选，仅日志则可省略 web）

### 包结构

| 类 | 说明 |
|----|------|
| `SchedulingApplication` | `@EnableScheduling` |
| `HeartbeatJob` | 每 5 秒 `fixedRate` 打日志 |
| `CounterController` | `GET /counter` 读取内存计数 |

### 运行与验证

```bash
cd springboot/sb-12-scheduling
mvn spring-boot:run
# 观察日志每 5 秒输出
curl http://localhost:8080/counter
```

## 练习

1. 使用 `cron` 仅在工作日 9 点执行。
2. 配置 `TaskScheduler` 线程池大小为 4。
