## 核心概念

### 定义

多线程共享进程资源，需同步保证数据一致性。优先使用线程池而非无限创建 `Thread`。

### 常见误区

- 认为 `volatile` 保证复合操作原子性
- 在锁对象上误用不同实例导致同步失效
- 线程池未关闭导致进程无法退出

## 关键 API / 注解

| 名称 | 说明 |
|------|------|
| `ExecutorService` | 线程池 |
| `synchronized` | 内置锁 |
| `CountDownLatch` | 等待多个任务完成 |
| `Callable` / `Future` | 有返回值任务 |
